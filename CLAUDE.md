# Morserino-32 NG -- Agent Instructions

## User Manual Maintenance

**`manual.md` in the project root must always reflect the current UI and feature set.**

When making changes that affect user-facing behavior -- adding/removing/renaming
modes, settings, menu items, key bindings, screens, status bar elements, web API
endpoints, or any control flow the user interacts with -- **update `manual.md`
accordingly in the same changeset.**

This includes but is not limited to:
- New or removed main menu entries
- Changes to `AppSettings` fields (new settings, changed ranges/defaults, removed settings)
- Changes to encoder/aux/paddle behavior or key mappings
- New or modified screens, modes, or games
- Web interface changes (new endpoints, UI features)
- Status bar indicator changes
- Power management or WiFi behavior changes
- Content type additions or modifications
